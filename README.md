# 🚀 دليل رفع وتشغيل ZipZap Rental CRM على GitHub Pages

هذا المستودع يحتوي على لوحة تحكم ونظام إدارة كامل لتأجير المناسبات متصل ومزامن بالكامل مع **Google Sheets (جداول بيانات جوجل)**.

---

## 🛠️ الخطوة 1: تهيئة شيت جوجل (Google Sheets)
1. افتح حسابك في [Google Drive](https://drive.google.com).
2. أنشئ شيت جوجل جديد باسم **`ZipZap CRM Master`**.
3. قم بإنشاء التبويبات (Tabs) التالية في الشيت (يدعم اللغتين العربية والإنجليزية):
   * `Orders` أو `الطلبات`
   * `OrderItems` أو `بنود الطلبات`
   * `Customers` أو `العملاء`
   * `Products` أو `المنتجات`
    * `Inventory` أو `المخزون`
    * `Suppliers` أو `الموردين`
    * `Pricing` أو `الأسعار`
    * `Summary` أو `ملخص`
    * `Employees` أو `الموظفين` (يُنشأ تلقائياً عند إضافة أول موظف لحفظ بيانات الموظفين وصلاحياتهم).
    * `ActivityLog` أو `سجل النشاطات` (يُنشأ تلقائياً لتسجيل وتتبع كافة نشاطات الموظفين والعمليات برقم المرجع لحظة بلحظة).

---

## 💡 المميزات الإضافية المتاحة:
* **🖨️ طباعة الفواتير:** كبسة زر داخل تفاصيل الطلب لطباعة إيصال أو فاتورة إيجار تفصيلية منسقة للعميل.
* **💬 إرسال الفواتير عبر الواتساب:** إمكانية إرسال تفاصيل الحجز والمنتجات والحسابات المالية بنص عربي منسق مباشرة للعميل على WhatsApp بضغطة زر.
* **📝 تتبع نشاطات الموظفين:** سجل متكامل يعرض في صفحة التقارير، يقوم بتسجيل أي إجراء (إضافة، تعديل، حذف، طباعة) بالوقت والمستخدم المسؤول لضمان الشفافية.

---

## 🔐 نظام صلاحيات الموظفين والدخول
* عند تشغيل النظام لأول مرة (أو في حال كان شيت الموظفين فارغاً)، يمكنك تسجيل الدخول باستخدام الحساب الافتراضي للمدير:
  * **اسم المستخدم (Username):** `admin`
  * **كلمة المرور / البين كود (Password/PIN):** `1234`
* **إدارة الموظفين والصلاحيات:**
  * بعد تسجيل الدخول كـ `admin`، ستظهر لك صفحة جديدة في القائمة الجانبية باسم **"إدارة الموظفين"**.
  * يمكنك من خلالها إضافة موظفين جدد وتحديد صلاحياتهم بدقة (أي الصفحات يمكنهم رؤيتها، وهل لديهم صلاحية تعديل وحذف وإضافة البيانات أم "عرض فقط").
  * الحسابات بنوع **"مدير (Admin)"** تمتلك كامل الصلاحيات تلقائياً وتستطيع إدارة شؤون الموظفين.

---

## 🔗 الخطوة 2: إعداد كود الـ API (Google Apps Script)
1. داخل شيت جوجل الرئيسي، اضغط على القائمة العلوية: **Extensions** ← **Apps Script**.
2. امسح أي كود موجود حالياً.
3. افتح ملف `GoogleAppsScript_API.txt` الموجود في هذا الفولدر، وانسخ كل الكود المكتوب بداخله والصقه في محرر الـ Apps Script.
4. اضغط على زر الحفظ (Save 💾).
5. اضغط على زر **Deploy** (في الأعلى باللون الأزرق) ← ثم اختر **New Deployment**.
6. اختر نوع التثبيت: **Web App**.
   * في خيار Execute as: اختر **Me (حسابك الشخصي)**.
   * في خيار Who has access: اختر **Anyone (أي شخص)** (هذا ضروري لكي تتمكن صفحة الويب من الوصول للشيت).
7. اضغط **Deploy** وامنح الصلاحيات اللازمة لحسابك (اضغط Advanced ثم Go to ZipZap CRM API (unsafe)).
8. بعد الاكتمال، انسخ رابط الـ **Web App URL** الناتج.

---

## 💻 الخطوة 3: ربط صفحة الويب بالـ API الخاص بك
1. افتح ملف `zipzap-crm.html` (وكذلك `index.html`) في أي محرر نصوص.
2. ابحث عن السطر رقم **412**:
   ```javascript
   const API_URL = 'رابط_الويب_أب_الخاص_بك_هنا';
   ```
3. استبدل الرابط الموجود بالرابط الذي نسخته من الخطوة السابقة.
4. قم بحفظ الملفات.

---

## 🌐 الخطوة 4: الرفع والتفعيل على GitHub Pages (مجاناً)
1. قم بإنشاء حساب على موقع [GitHub](https://github.com) إذا لم يكن لديك حساب.
2. أنشئ مستودعاً جديداً (New Repository) وسمّه مثلاً `zipzap-crm`.
3. ارفع الملفات التالية إلى المستودع:
   * `zipzap-crm.html` (قم بتغيير اسمه إلى `index.html` لكي يفتح تلقائياً كصفحة رئيسية).
   * `README.md`
4. بعد رفع الملفات بنجاح، اذهب إلى إعدادات المستودع **Settings** من الشريط العلوي في GitHub.
5. من القائمة الجانبية اليسرى اذهب إلى خيار **Pages**.
6. تحت قسم **Build and deployment** (Build and deployment Source):
   * اختر الفرع (Branch) ليكون **`main`** أو **`master`** بدلاً من `None`.
   * اختر المجلد الفرعي ليكون **`/ (root)`**.
7. اضغط على زر **Save**.
8. انتظر دقيقة واحدة، ثم قم بتحديث الصفحة وسوف يظهر لك رابط موقعك المباشر في الأعلى!
